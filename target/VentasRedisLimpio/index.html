Simulador de Venta con Caché Redis

Frontend con HTML y JavaScript (AJAX) para comunicarse con el Servlet.

¿Qué sucede "detrás de cámaras"?

Observe la consola de salida de NetBeans Y la pestaña "Network" de las herramientas de desarrollador de su navegador.

  1. CACHE MISS: La primera vez que se compra, el sistema busca en MySQL y guarda en Redis.
  2. CACHE HIT: Las siguientes consultas del mismo producto se obtienen de Redis.
  3. CACHE INVALIDADA: Tras una venta, la copia en Redis se borra para mantener la consistencia.
